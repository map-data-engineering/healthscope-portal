Methodology

How we source, clean, harmonise, and quality-assess health facility data

Overview

This page describes the end-to-end pipeline used to produce each country dataset. The pipeline runs through four stages: source collation → harmonization → quality assessment → versioned release. All pipeline code is open-source and available in the GitHub repository.

1. Source Collation

Primary Sources

Uganda MoH Uganda HMIS Annual

Secondary Sources

Where the MFL has coverage gaps we supplement with DHIS2 organisation unit exports, OpenStreetMap for coordinate cross-validation, and HDX historical snapshots (e.g. the WHO AFRO Ethiopia coordinate compilation).

2. Harmonization

Common Schema

All datasets are transformed to a shared flat schema before analysis:

facility_id     character   Portal-assigned unique ID (ISO2_NNNNN)
source_id       character   Identifier in the original source
name            character   Facility name (original language)
facility_type   character   Hospital / Health Centre / Dispensary / Clinic / Other
ownership       character   Public / Private-for-profit / Faith-based / NGO
admin1          character   Province / Region
admin2          character   District / County
admin3          character   Division / Sub-county (if available)
admin4          character   Ward / Parish (if available)
longitude       numeric     WGS84 decimal degrees
latitude        numeric     WGS84 decimal degrees
coord_source    character   GPS / Geocoded / Centroid / Unknown
operational     logical     Operational at time of data extraction
year            integer     Reference year of the record
version         character   Portal dataset version

3. Quality Assessment

📍 Coordinate Validity

Proportion of records with non-missing GPS coordinates falling within the country boundary. Centroid-imputed coordinates are flagged separately.

🗂 Attribute Completeness

Proportion of non-missing values per schema field, reported individually and as an overall completeness score (0–100).

🔁 Duplicate Rate

Estimated proportion of duplicate records before and after deduplication using the name × coordinate matching procedure.

📅 Temporal Coverage

Year range spanned by records, and the proportion of facilities with a recorded update date within the past 24 months.

🏛 Administrative Consistency

Proportion of records whose admin1–admin2 codes are consistent with official GADM v4.1 administrative boundary files.

4. Versioned Release

Datasets follow semantic versioning (MAJOR.MINOR.PATCH):

  • MAJOR — schema-breaking changes
  • MINOR — new facilities or additional fields (backward-compatible)
  • PATCH — coordinate corrections or name fixes
